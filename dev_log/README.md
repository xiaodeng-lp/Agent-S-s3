# 开发日志规范

## 文件结构

```
dev_log/
  README.md                    ← 本文件（规范）
  INDEX.md                     ← 时间倒序索引（每次新增日志时更新）
  YYYY-MM/
    MMDD_作者_主题.md           ← 日志文件
```

## 文件命名

```
dev_log/YYYY-MM/MMDD_作者_主题.md
```

- `YYYY-MM`：年月目录，同月日志归档到同一目录
- `MMDD`：月日（两位数）
- `作者`：开发者标识（英文名或拼音缩写，如 `hu_ray`、`zhang_san`）
- `主题`：简短英文标识（如 `baseline`、`reflection_fix`、`anchor_store`）

示例：
```
dev_log/2026-05/0504_hu_ray_baseline.md
dev_log/2026-05/0506_zhang_reflection_fix.md
```

## 日志格式

每篇日志包含以下章节：

```markdown
# <标题> — YYYY-MM-DD

作者：<作者名>

## 背景
- 本次开发的出发点和目标

## 变更清单
| 文件 | 操作 | 说明 |
|------|------|------|

## 关键决策
- 决策 + 理由

## 验证
- [ ] 检查项
- [ ] 检查项

## 风险 & 后续
- 已知风险
- 下一步计划
```

## INDEX.md 格式

```markdown
# 开发日志索引

| 日期 | 作者 | 主题 | 文件 |
|------|------|------|------|
| 2026-05-04 | hu_ray | Feishu CUA 基线 | `2026-05/0504_hu_ray_baseline.md` |
| 2026-05-04 | hu_ray | 反射频率控制 | `2026-05/0504_hu_ray_reflection.md` |
```

最新日志排在最前。

## 维护规则

- 每次有意义的改动后新增日志，并同步更新 INDEX.md
- 不记录琐碎的 lint/格式修改
- 同一天同一作者的多个独立改动，拆分为多篇日志（不同主题）
- 每篇日志对应一个或多个 git commit
- 多人同一天开发各写各的日志，按作者区分
